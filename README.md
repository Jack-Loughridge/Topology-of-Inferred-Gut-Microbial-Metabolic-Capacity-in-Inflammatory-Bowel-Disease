# Species-abundance benchmarks: all five tasks

GitHub-ready implementation of the species-abundance comparator models for the five locked IBDMDB classification tasks:

1. IBD vs non-IBD
2. non-IBD vs UC vs CD
3. non-IBD vs UC
4. non-IBD vs CD
5. UC vs CD

For every task, the package runs L1 logistic regression, random forest and XGBoost over **20 repetitions × 5 participant-grouped folds**. It consumes the exact task manifests generated by the joint H0+Ricci all-task preflight and never regenerates folds independently.

## Method

Each fold uses only outer-training samples to fit:

- the CLR transformation convention and pseudocount;
- variance filtering;
- standardisation;
- class-balanced sample weighting.

The fixed benchmark models are:

- L1 logistic regression: `C=0.2`, SAGA, `max_iter=10000`;
- random forest: 1000 trees, `min_samples_leaf=3`, `max_features="sqrt"`;
- XGBoost: 700 trees, depth 3, learning rate 0.02, subsample 0.85, column subsample 0.85, L2=2.0 and L1=0.25.

For binary XGBoost, imbalance is handled once through balanced sample weights; `scale_pos_weight=1.0`.

Canonical species rows that are all zero after taxonomic filtering are excluded explicitly. In the current cohort these correspond to samples whose original profile is `UNKNOWN=100%`. Every retained sample preserves its locked repetition, fold and train/test role.


## Principal outputs

Each task directory contains pooled sample- and participant-level OOF metrics, fold metrics, consensus predictions, confusion matrices, calibration outputs, model diagnostics, feature-stability tables and optional full-source models.

The combined result table is:

```text
aggregate/repetition_performance_summary_all_tasks.csv
```

Use:

```bash
python3 show_results.py
```

## Reproducibility and resume

Every run is locked to the scientific configuration, source file and manifest hashes. Completed model-fold artifacts are resumed without refitting. A non-empty incompatible output directory is rejected unless explicitly overwritten.

Microbiome data, split outputs, fitted models and result directories are not included in this repository.
